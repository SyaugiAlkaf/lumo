AMANAH — A TRUST FAITHFULLY KEPT

Amanah

The agent reads. The chain enforces.

An on-device AI treasury agent for cross-border SME payments in USDC on Stellar — the model only reads invoices; deterministic policy and two Soroban contracts move the money.

Programmable trust, enforced on-chain.

A policy-signer smart account refuses out-of-policy transactions; a conditional escrow can pay only the bound supplier — or refund the SME. No third path exists.

Any chain, any code — SDK, REST, MCP.

Python SDK, REST with OpenAPI, an MCP server for any agent. Settlement sits behind adapter seams — Soroban today, EVM next.

Before / after.

Before Amanah
  • 3–5 day correspondent banking
  • letters of credit — trust by paperwork
  • an AI agent holding keys can be tricked into paying the wrong party
After Amanah
  • instant USDC on Stellar
  • policy-enforced, on-chain
  • the AI can be tricked — the money cannot

Proven on Stellar testnet.

Deployed live and settled end-to-end — escrow funded, shipment attested, 100 test USDC released to the bound supplier only.
